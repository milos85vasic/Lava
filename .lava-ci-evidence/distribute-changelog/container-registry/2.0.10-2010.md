lava-api-go 2.0.10 (2010) — 2026-05-05

What's new vs 2.0.9 (2009):

ADDED:
- Server-side Firebase telemetry middleware at `internal/middleware/firebase.go`.
  * Records HTTP 5xx responses + recovered panics as Firebase non-fatals (when LAVA_FIREBASE_ADMIN_KEY is configured) OR as structured-log entries (no-op fallback when no service-account key).
  * Logs 4xx + 2xx as events for traffic accounting.
  * Wired into `cmd/lava-api-go/main.go` `buildRouter` so production traffic gets observed.
  * 6 unit tests with falsifiability rehearsal — see `internal/middleware/firebase_test.go`.
- Server-side Firebase client at `internal/firebase/firebase.go` — Admin SDK skeleton with honest no-op fallback when no service-account key.

CONFIGURATION (operator action):
- To enable Firebase Admin SDK reporting, generate a service-account JSON in the Firebase Console (Project Settings → Service Accounts → Generate new private key) and save as `lava-api-go/firebase-admin-key.json` (gitignored). Set `LAVA_FIREBASE_ADMIN_KEY=/path/to/key.json` and `LAVA_FIREBASE_PROJECT_ID=lava-vasic-digital` in the service environment.
- Without the key the middleware degrades to structured-log forwarding — server stays online, traffic stats logged.
