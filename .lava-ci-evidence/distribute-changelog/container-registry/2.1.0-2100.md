WHAT'S NEW

API-side auth:
- AuthMiddleware enforces the Lava-Auth header (name itself config-
  driven via LAVA_AUTH_FIELD_NAME per §6.R). Active hash → 200; retired
  hash → 426 + min-version JSON; unknown / missing / malformed → 401
  + per-IP backoff counter advance.
- BackoffMiddleware sits in front of AuthMiddleware: returns 429 +
  Retry-After when the per-IP fixed-step ladder has the client blocked.
- Per-IP fixed-ladder backoff via Submodules/RateLimiter/pkg/ladder
  (upstream contribution; pinned at 3faf7a51). Default ladder:
  2s, 5s, 10s, 30s, 1m, 1h. Configurable via LAVA_AUTH_BACKOFF_STEPS.
- Trusted-proxy CIDR honor (LAVA_AUTH_TRUSTED_PROXIES); X-Forwarded-For
  unwrap for hosts in the trusted set, RemoteAddr authoritative
  otherwise.

Transport policy:
- HTTP/3 preferred with HTTP/2 fallback. The HTTP/2 listener emits
  Alt-Svc: h3=":<port>"; ma=86400 so HTTP-3-capable clients upgrade.
- Brotli response compression on Accept-Encoding: br.
- Prometheus counter lava_api_request_protocol_total{protocol,status}
  for HTTP/3 adoption tracking.

Constitutional addition:
- §6.R No-Hardcoding Mandate added to root CLAUDE.md, AGENTS.md, and
  all 16 Submodules/*/CLAUDE.md per §6.F inheritance. Pre-push grep
  enforces (UUID literals, plus targeted enforcement of additional
  literal classes in future phases).
