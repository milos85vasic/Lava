lava-api-go 2.0.16 (2016) — 2026-05-06

What's new vs 2.0.15 (2015):

FIX (post-Ktor cleanup, §6.J bluff in lava-containers CLI):

The lava-containers workstation CLI had three commands (build, status,
logs) that silently targeted dead surfaces post-Ktor-:proxy-removal:

- `-cmd=build` shelled `podman build -t ... ./proxy`. The ./proxy
  directory was deleted in 2.0.12 (commit a00b28f); this command
  would fail at runtime.
- `-cmd=status` probed http://localhost:8080/. The api-go service
  listens on https://localhost:8443/health. Status always reported
  Healthy: false even when api-go was running locally.
- `-cmd=logs` called `podman logs lava-proxy`. The lava-proxy
  service no longer exists in docker-compose.yml; the active
  service is lava-api-go.
- internal/runtime/runtime.go IsHealthy()+ContainerIP() filtered
  on name=lava-proxy. Replaced with LavaContainerName="lava-api-go".
