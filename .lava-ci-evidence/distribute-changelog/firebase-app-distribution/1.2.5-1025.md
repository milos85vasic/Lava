Lava-Android 1.2.5 (1025) — 2026-05-05

What's new vs 1.2.4 (1024):

FIXED (preemptive Hilt-graph hardening, post-§6.O cycle):
- The 1.2.3 → 1.2.4 fix hardened the LavaApplication Firebase init path but NOT the Hilt graph. If FirebaseAnalytics.getInstance() / FirebaseCrashlytics.getInstance() / FirebasePerformance.getInstance() threw on a particular device, the next ViewModel that injects AnalyticsTracker (Login, ProviderLogin, Search, Topic) would crash at construction.
- 1.2.5 closes this gap:
  * FirebaseProvidesModule wraps each SDK accessor in runCatching{...}.getOrNull() — null SDK is acceptable.
  * FirebaseAnalyticsTracker accepts nullable SDKs and runCatching-guards every per-call SDK invocation.
  * NoOpAnalyticsTracker is the fallback when both Crashlytics and Analytics are unavailable — every call becomes a logcat warning instead of crashing the consumer.
  * 4 new JVM unit tests cover the nullable-SDK path, the throwing-SDK path, and the present-SDK-forwarding path.

ADDED:
- §6.P (Distribution Versioning + Changelog Mandate) enforcement extended to scripts/tag.sh — refuses tags lacking CHANGELOG.md entry or per-version snapshot.
- Bluff-hunt evidence record at .lava-ci-evidence/bluff-hunt/2026-05-05-firebase-and-distribute-mandates.json (5 rehearsals + 2 production-code targets per §6.N.2).
- Per-version distribute-changelog snapshots for proxy (1.0.5-1005) + api-go (2.0.10-2010) channels — needed because §6.P requires them per channel.
