## Lava-Android-1.2.26-1046 / Lava-API-Go-2.3.15-2315 — 2026-05-17 (Bug 2 3-layer cascade fix — anonymous-only search now works)

**Previous published:** Lava-Android-1.2.25-1045 (debug + release distributed 2026-05-17 evening).

### Bug 2 FIXED + §6.J-verified on Pixel_8/API35

Anonymous-only multi-provider search (the operator's "We have onboarded 2 providers who do not require credentials. When we have tried to search we get error!") is now fully working. Root cause was a 3-layer cascade in production code:

1. **OnboardingViewModel.onTestAndContinue** never wrote to `provider_configs` after `sdk.switchTracker`/`sdk.login` — fixed via `providerConfigRepository.ensureDefault(currentId)` in both anon + cred paths.
2. **SearchInputViewModel.onSubmit** had a race where `selectedProviders` could be empty if user submitted faster than the Bug 3 fix's `viewModelScope.launch` repo-read — fixed via blocking `resolveProviderIdsForSubmit()` suspend helper.
3. **SearchResultNavigation** dropped `providerIds` entirely from both serializer (`openSearchResult`) and deserializer (`SavedStateHandle.filter`) — fixed via new `ProviderIdsKey = "pids"` constant wired into both.

§6.J falsifiability evidence: same emulator reproduced "Error" pre-fix and "Nothing found" + correct chip-bar post-fix (UI XML dumps preserved at `/tmp/lava-bug2-repro/`).
