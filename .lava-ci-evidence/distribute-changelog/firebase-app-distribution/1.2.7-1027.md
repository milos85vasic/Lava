WHAT'S NEW

Auth foundation:
- Every API request now carries the per-build encrypted UUID in a
  Lava-Auth-style header (name from .env, not source).
- AuthInterceptor decrypts on each request via AES-256-GCM keyed by
  HKDF-SHA256(salt = SHA256(signing-cert)[:16], ikm = pepper). The
  plaintext UUID lives only as ByteArray inside intercept(),
  fill(0)'d in finally. Per core/CLAUDE.md memory hygiene.
- Build-time codegen task (generateLavaAuthClass{Debug,Release}) reads
  .env + the variant keystore and emits the encrypted blob into
  app/build/generated/lava-auth/main/lava/auth/LavaAuthGenerated.kt.
  Generated dir is gitignored; regenerated each build.
- Re-signed-APK fail-closed: a re-signed APK has a different cert
  hash → different derived key → AES-GCM decrypt fails with
  AEADBadTagException → AuthInterceptor surfaces it as IOException
  rather than letting a malformed header reach the API.

α-hotfix:
- TrackerDescriptor.apiSupported gates the user-facing provider list:
  ProviderLoginViewModel + TrackerSettingsViewModel filter on
  `verified && apiSupported`. Internet Archive (and other providers
  without lava-api-go routes today) are hidden from onboarding until
  Phase 2 ships per-provider routing. Closes the alice-bug class.
